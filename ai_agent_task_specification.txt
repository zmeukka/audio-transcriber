Ты технический писатель. Твоя задача — создать подробное техническое задание для разработки программы, которая конвертирует файлы со звуком в текст при помощи whisperX.
Ты должен помнить что заказчик не програмист и может не знать некоторые тонкости программирования.

Ты должен создать техническое задание со следующими пунктами:

- Программа должна быть написана на языке Python.
- Все комментарии в коде должны быть на английском языке.
- У программы должен быть файл настроек в формате YAML, в котором можно будет указать:
  - Путь к входной папке с аудиофайлами.
  - Путь к выходной папке для сохранения текстовых файлов.
  - Язык распознавания речи (например, английский, русский).
  - Список ID telegram пользователей, для которых известно кто говорит в аудиофайлах. Так же должны быть различные алиасы которые привязаны к данному ID (например имя, кличка и прочее).

- Программа должна перед обработкой файлов создать в выходной папке два файла с результатами обработки:
  - Файл "*.in_process", в котором сохраняется расшифровка договора. * - это имя оригинального аудиофайла без расширения. Обычно это telegramID+salt.
  - Файл "*.errors", в котором будут сохраняться ошибки, возникшие при обработке файла.

- При обнаружении необработанного файла (файла, для которого нет файла .in_process или .errors) программа должна начать его обработку.

- При обнаружении файла .in_process программа должна продолжить обработку с того места, на котором она остановилась. Если обработка невозможна то надо записать это в .error файл. Так же надо сохранять количество попыток, если количество обработок больше трех и нет success оригинальный файл и файлы созданные программой переносятся в другоую директорию для ручной обработки.
